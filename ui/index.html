Как пользоваться vizo

Краткий гайд по всем разделам. Если что-то не работает — открой Настройки и проверь API-ключи.

📋
Сценарии

Сценарий — это собранная цепочка шагов: получить URL → транскрипт → отправить в Claude → озвучить → сохранить. Один сценарий = один пайплайн.

Как создать:

  1. Главное меню → Сценарии → «+ Создать сценарий»
  2. Слева — библиотека типов шагов (YouTube, Claude, Файлы, Озвучка, Картинки, Видео)
  3. Клик по типу — шаг добавляется в цепочку справа
  4. Раскрой шаг (шеврон ▾) и заполни параметры
  5. «Сохранить» → «Запустить»

Готовые сценарии: в списке уже есть встроенные — Тартария, Перевод на польский, Только озвучка. Можно их редактировать или дублировать как стартовую точку для своих.

Переменные: каждый шаг кладёт результат в переменную (например {transcript}). Следующие шаги могут её использовать в шаблонах промптов, имени файла, источнике текста.

Сессии Claude: шаг «Открыть новый чат Claude» начинает новую сессию. Все «Отправить промпт» / «Спросить пользователя» после него работают в одном чате. Чтобы открыть свежий чат без памяти — добавь ещё один «Открыть новый чат».

Параллельный шаг: через меню «⋮» на шаге выбери «Параллельно с предыдущим» — он будет выполняться одновременно с предыдущим. Удобно для озвучки + картинок.

🌐
Добавить новый язык перевода
  1. Сначала добавь голос для этого языка: Настройки → Голоса → «+ Создать голос». Выбери движок: Озвучка Матея (укажи voice_id и модель) или VoiceBot (укажи только UUID готового шаблона).
  2. Потом: Настройки → Языки перевода → «+ Добавить язык»
  3. Введи название («Испанский»), выбери флаг из сетки, привяжи голос, по желанию напиши свой промпт-шаблон
  4. Сохрани — язык появится на экране «Перевод»
🎙
Голоса

Настройки → Голоса. По умолчанию 4 встроенных (Тартария, Польский, Венгерский, Чешский), можно добавить свои.

Тест прослушивания: в редакторе голоса справа — кнопка «🔊 Прослушать». Сначала укажи Voice API ключ в настройках.

Удалить встроенный: можно. Если будет нужен — Настройки → «Восстановить встроенные пресеты».

🔑
API-ключи

VeoNonStop API Key — нужен для генерации картинок (Banana 2/Pro) и видео в сценариях.

Voice API Key — нужен для озвучки и теста голосов. Две озвучки, два ключа: Озвучка Матея (Bearer-токен от voicer.mat3u.com) и VoiceBot (X-API-Key от voiceapi.csv666.ru).

Claude — работает через автоматизацию браузера, нужно один раз «Привязать Claude аккаунт» в настройках (откроется Chrome, авторизуйся в claude.ai и закрой окно).

🏛
Тартария как пример

Открой Сценарии → Тартария, чтобы посмотреть как устроена сложная цепочка из 11 шагов: получение видео, две сессии Claude с диалогом и загрузкой картинок-примеров, параллельная озвучка и генерация ~100 картинок.

Можно изменить промпты прямо в редакторе шагов или дублировать сценарий и сделать свою версию.

🧩
Типы шагов — что делает каждый

Шаги в библиотеке редактора сценариев слева. Сгруппированы по категориям. Каждый кладёт результат в именованную переменную, которую следующие шаги используют через {имя}.

📥 Входы

YouTube URL — спрашивает у тебя ссылку на видео при запуске. Кладёт URL в переменную (по умолчанию {url}). Обычно первый шаг сценария.
Текст — спрашивает произвольный текст при запуске. Полезно для сценариев «только озвучка» — пишешь текст, шаг кладёт его в {text}, дальше озвучка использует.

🌐 YouTube

Заголовок — забирает заголовок видео из YouTube. Вход: {url}. Выход: {title}.
Описание — описание под видео. Вход {url}, выход {description}.
Транскрипт — автоматические субтитры видео. Вход {url}, выход {transcript}. Если у ролика нет субтитров — шаг упадёт с ошибкой.
Превью — скачивает миниатюру (thumbnail.jpg) видео в папку результатов. Вход {url}, выход — путь к файлу.

🤖 Claude

Открыть новый чат — открывает свежий чат Claude в браузере. Сбрасывает контекст предыдущей сессии. Обязательный шаг перед любым «Отправить промпт» или «Спросить пользователя».
Отправить промпт — посылает текст в активный чат, ждёт ответа Claude. Поле «Шаблон промпта» — текст с подстановками {переменная}. Опционально можно прикрепить файлы (картинки-примеры). Выход — ответ Claude в указанную переменную.
Перевод/адаптация по кускам — для длинных текстов, которые Claude не осилит за один ответ. Сначала шлёт весь оригинал + твои правила (поле «Вступительный промпт», плейсхолдер {original}), потом сам режет текст на куски по концам предложений (размер на выбор: ≈2–3к / 5–7к / 10–15к символов) и отправляет их по одному в тот же чат. Ответ по каждому фрагменту сразу дописывается в один txt (через пустую строку), а склейка всех ответов кладётся в выходную переменную. Перед шагом нужен «Открыть новый чат».
Спросить пользователя — приостанавливает сценарий. Показывает тебе последний ответ Claude и ждёт пока ты введёшь свой текст в UI. Твой текст уходит в чат как следующее сообщение, ответ Claude кладётся в переменную.
Закрыть чат — явно завершает текущую сессию Claude. Не обязателен — сессия закроется автоматически в конце сценария или при следующем «Открыть новый чат».

💾 Файлы

Сохранить TXT — записывает переменную в текстовый файл в папке результатов. Поля: «Источник» — имя переменной (например script) и «Имя файла» (script.txt).
Сохранить JSON — то же что TXT, но если в переменной dict/list — запишется как форматированный JSON.

🎙 Озвучка

Синтез голоса — генерит mp3 из переменной через выбранный в голосе движок (Озвучка Матея или VoiceBot). Поля: «Голос» — выпадающий список из настроек, «Источник» — имя переменной с текстом, «Имя файла» — куда сохранить. Нужен соответствующий Voice API ключ в настройках. Во время работы пишется статус задачи (для VoiceBot — прямо из API).

🖼 Картинки (VeoNonStop)

Banana — одна картинка — генерит 1–8 картинок по одному промпту. Модель: 🍌 Banana 2 (быстрая) или ★ Banana Pro (качественнее). Соотношение 16:9 / 9:16 / 1:1 / 4:3 / 3:4. Промпт может содержать {переменные}. Выход — список путей к файлам в папке images/.
Banana — список из переменной — берёт переменную со списком промптов (каждая строка = один промпт = одна картинка). Идеально для «100 промптов от Claude → 100 картинок». Поле «Источник промптов» — имя переменной (например image_prompts).

🎬 Видео (VeoNonStop)

Text → Video — генерит видео из текстового промпта. Длительность 4s / 6s / 8s. Кол-во 1–4. Соотношение 16:9 / 9:16.
Image → Video — анимирует картинку (нужно указать путь). Тот же набор параметров.
Batch Frame — генерит видео-переход между двумя кадрами (стартовый → конечный). Нужно указать два пути картинок и описание перехода.

Параллельное выполнение: любому шагу через меню «⋮» можно поставить флаг «Параллельно с предыдущим». Тогда движок будет выполнять текущий шаг одновременно с предыдущим — экономит время на независимых задачах (например озвучка одновременно с генерацией картинок).
