# vizo

Десктоп-приложение для автоматизации YouTube-видео: транскрипт → адаптация/перевод через Claude → озвучка → картинки/видео. Сценарии настраиваются внутри приложения, запуски идут через очередь (можно ставить несколько на ночь).

---

## Скачать / установить

Готовые сборки — на вкладке **[Releases](../../releases)**.

### Windows
1. Скачайте `vizo.exe` из последнего релиза.
2. Запустите. Всё — настроек установки нет.
3. При первом запуске войдите в Claude в открывшемся окне браузера и впишите API-ключи в **Настройках**.

### macOS (экспериментально)
1. Скачайте `vizo-macos.zip`, распакуйте.
2. Сборка **не подписана**, поэтому macOS покажет предупреждение. Первый запуск:
   правый клик по `vizo.app` → **Открыть** → **Открыть**. (Или в Терминале: `xattr -cr vizo.app`.)

## Обновления

В приложении: **Настройки → Проверить обновления**. Кнопка смотрит последний релиз на GitHub, сравнивает версию и предлагает обновиться в один клик (Windows). Поэтому новый релиз = обновление сразу у всех пользователей.

---

## Разработка

```bash
pip install -r requirements.txt
python -m playwright install            # драйверы Playwright (браузер берётся системный Chrome)
python app.py
```

Требуется установленный **Google Chrome** (Claude автоматизируется через реальный Chrome с отдельным профилем).

Структура — см. `PROJECT_CONTEXT.md`.

---

## Сборка .exe (локально, Windows)

```bat
build.bat
```
Собирает `build-nuitka\vizo.exe` через Nuitka (10–20 мин). Версия берётся из `config.py` (`VERSION`).

## Как выпустить обновление

Версия в одном месте — `config.py` → `VERSION`. Релизный тег должен быть **больше** установленной у пользователей версии, иначе кнопка обновления не сработает.

**Вариант A — локально (Windows, самый надёжный):**
1. Поднимите `VERSION` в `config.py` (напр. `1.1.0` → `1.2.0`).
2. Закоммитьте и запушьте.
3. Запустите `release.bat` — он соберёт `vizo.exe` и опубликует релиз `vX.Y.Z` через `gh`.

**Вариант B — GitHub Actions (Windows + macOS автоматически):**
1. Поднимите `VERSION` в `config.py`, закоммитьте, запушьте.
2. Поставьте тег и запушьте его:
   ```bash
   git tag v1.2.0
   git push origin v1.2.0
   ```
3. CI соберёт Windows (Nuitka) и macOS (PyInstaller) и приложит обе сборки к релизу.
   > CI-сборка Mac не подписана и может потребовать доводки при первом запуске воркфлоу.

## Безопасность / что НЕ коммитится

API-ключи и сессии в репозиторий не попадают (см. `.gitignore`): `user_settings.json`, `claude_auth.json`, `chrome_profile/`. Ключи задаются каждым пользователем в Настройках приложения.
